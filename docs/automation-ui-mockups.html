What each variation is trying to prove

  • A (Presets): Is a curated gallery enough? Fewest decisions, but you can only get the looks we ship.
  • B (Shape + 2 sliders): The middle ground — simple by default, full control one click away under “Advanced”.
  • C (Inline strip): Does removing the modal entirely feel better for a fast live workflow? Trades preview size for density.
  • D (Preset stepper): Variation A made compact — same presets, cycled one at a time with ‹ › / arrow keys. One big preview instead of a 3×3 grid; least scrolling.

Shared principles baked in (these also fix the bugs you hit)

  • Slow + time-based by default — Speed starts low; one full cycle every several seconds. Beat/BPM is opt-in only (B).
  • The live value visibly animates — the “value” bar/number under each preview moves continuously, so an active automation is obviously alive (today the param UI sits still).
  • Remove restores the base — “Remove” snaps the param back to its manual value rather than freezing it at the last formula output.
